# Карта сценария «Тень дома»

## Поток пользователя

```text
вход → мечта → интро → акт 1 (тени 1–3) → [проверка реальности при необходимости]
     → акт 2 (тени 4–6) → [телесный блок, если есть сильный отклик]
     → акт 3 (тени 7–10) → блок глубины → выявление противоречий
     → паспорт тени → микро-разрешение → продажный переход
```

## Что хранится после каждого шага

### 1. Старт сессии
- session_id
- created_at
- status = `active`

### 2. Мечта
- `dream`
- длина ответа
- пример мягкой подсказки, если ответ слишком короткий

### 3. Каждая тень
- выбранный вариант A / Б / В
- текстовое уточнение
- балл (A=1, Б=2, В=3)
- `strong_signal`
- `body_markers`, если уже спрашивали тело
- `bridge_target_candidates`

### 4. Проверка реальности
Запускается, если в первых тенях много ответов `В`, но нет живого материала.
Храним:
- что пользователь начал за 3 месяца;
- что довел до результата;
- где все еще откладывает.

### 5. Телесный блок
Храним:
- body_signal: сжатие / напряжение / тяжесть / хочется закрыться / хочется отступить / другое
- body_text

### 6. Блок глубины
Храним:
- где был самый сильный отклик;
- реальные ситуации;
- чего человек избегает;
- от чего это защищает;
- какую цену он уже платит.

### 7. Противоречия
Храним:
- contradictions_detected[]
- contradiction_text
- user_explanation

### 8. Паспорт тени
Храним:
- top_shadows[]
- mechanism_formula
- manifestation
- price
- hidden_resource
- screen_phrase
- micro_permission
- offer_text

## Где вызывается ИИ

ИИ вызывается точечно:
1. короткая живая реакция после ответа;
2. мягкое углубление / bridge-вопрос;
3. «предсказание человека»;
4. микровывод в блоке глубины;
5. финальный «Паспорт тени»;
6. продающий переход в «Чертеж дома».

## Где работают правила без ИИ

Правилами управляется:
- порядок шагов;
- подсчет баллов;
- флаг сильного отклика;
- запуск телесного блока;
- запуск проверки реальности;
- поиск базовых противоречий;
- выбор топ-1 и топ-2 теней;
- финальный маршрут сессии.
